# Tabel — учёт посещаемости и расчёт зарплаты

Внутреннее веб-приложение для учёта посещаемости сотрудников и расчёта зарплаты по двум периодам месяца. До 50 пользователей, self-hosted.

**Роли:** `FOREMAN` (бригадир) — заполняет табель своей бригады; `ADMIN` — сводный табель, утверждение, справочники; `DIRECTOR` — то же что админ.

**Бизнес-правила:**
- Значения дня: `FULL` (1) / `THREE_QUARTERS` (¾) / `HALF` (½) / `QUARTER` (¼) / `ABSENT` (0) / `SICK` (б).
- В зарплату идут только смены: 1, ¾, ½, ¼. Больничный и прогул = 0.
- Период 1: дни 1–15, период 2: дни 16–конец месяца. Сумма смен × ставка.
- Статусы табеля: `DRAFT → SUBMITTED → APPROVED`. После APPROVED правит только админ.
- Изоляция: бригадир видит только свой табель.
- Снимок ставки и должности фиксируется в `timesheet_rows` — историчность.
- Один сотрудник может быть в разных бригадах в разные месяцы; конфликт за один месяц — предупреждение, не блокировка.

## Стек

- Next.js 16 (App Router) + TypeScript + Server Actions
- PostgreSQL 16 + Prisma + Alembic-стиль миграций (`prisma migrate`)
- NextAuth (Credentials) + bcrypt
- Tailwind 4 + shadcn/ui + TanStack Table
- ExcelJS для экспорта XLSX
- Docker Compose для локального Postgres и production-деплоя

## Команды


## Архитектурные конвенции

- Бизнес-логика и расчёты — **только в `src/lib/`**. UI ничего не считает.
- `src/lib/payroll.ts` — единственный источник истины для расчёта зарплаты.
- `src/lib/excel.ts` — единственное место генерации XLSX.
- `src/lib/rbac.ts` — все проверки доступа по ролям; ни одна страница/server action не делает ручную проверку роли.
- Server Actions для мутаций; читать через серверные компоненты.
- Снимки (`*_snapshot`) при создании строки табеля — никогда не пересчитывать ставку из текущего справочника.
